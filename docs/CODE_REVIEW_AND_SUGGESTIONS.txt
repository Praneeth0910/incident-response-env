BUG #5 — SINGLE GLOBAL ENV INSTANCE IN server/app.py IS RECREATED ON EVERY /reset
------------------------------------------------------------------
File: server/app.py, lines ~55-66

The code does:
  _env: IncidentResponseEnv = IncidentResponseEnv()
  
  @_app.post("/reset")
  async def reset(body):
      global _env
      _env = IncidentResponseEnv()   # ← creates a new instance every time
      obs = _env.reset(...)

But dashboard_impl.py's create_dashboard() takes env_instance as a parameter
and closes over it. When /reset creates a new _env, the dashboard still holds
a reference to the OLD env object. So the Gradio dashboard's state and the
REST API's state diverge after the first reset from the API.

IMPACT: Medium — if a judge uses the API directly AND the dashboard in the
same session, they'll see different state. Not a hard fail but looks sloppy.

FIX: Instead of replacing the instance, call reset on the existing one:

  @_app.post("/reset")
  async def reset(body):
      try:
          task_id = body.task_id if body else "task_cpu_spike"
          seed = body.seed if body else None
          obs = _env.reset(task_id=task_id, seed=seed)  # reuse, don't replace
          return obs.model_dump()
      except KeyError:
          raise HTTPException(status_code=400, detail=f"Unknown task_id: {task_id}")

PROMPT FOR CLAUDE CODE:
"In server/app.py, remove the line '_env = IncidentResponseEnv()' inside
the reset() endpoint handler. Instead, call '_env.reset(task_id=task_id,
seed=seed)' directly on the existing global _env. Remove the 'global _env'
declaration since it's no longer needed. This ensures the dashboard and the
REST API share the same environment instance."
